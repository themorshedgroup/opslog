# TMG Ops Log — Training Guide & SOP **Version:** 1.1 | **Last Updated:** May 2026 **For:** The Morshed Group — All Team Members --- ## 1. What Is TMG Ops Log? TMG Ops Log is an internal operations tracking tool for The Morshed Group real estate team. It allows the team to manage tasks, log updates, track decisions, and generate end-of-day (EOD) summaries — all in one place, powered by AI. **Who uses it:** - **Symon (Operations Manager)** — Primary user. Manages all tasks, logs daily progress, generates EOD summaries for Tarek. - **Tarek (CEO)** — Receives EOD notifications with task updates and decision requests. - **Other team members** — May view tasks, add thread updates, or log accomplishments. **How to access:** Open the app in any web browser. On mobile, you can save it to your home screen as a PWA (tap Share → Add to Home Screen). --- ## 2. Getting Started ### First-Time Setup 1. Open the app and go to the **⚙️ Settings** tab 2. Enter your **Anthropic API Key** in the field (starts with `sk-ant-...`) 3. Tap **Save** — you'll see a green "✓ Key saved" confirmation 4. Select your preferred **AI model** (Claude Sonnet 4 is recommended for daily use) 5. Toggle **Dark Mode** on/off to your preference > **Note:** Without an API key, the Board and Threads tabs still work, but AI features (chat, EOD, Agenda, AI Suggest) will be disabled. --- ## 3. Navigating the App The app has **5 tabs** at the bottom of the screen: | Tab | Icon | Purpose | |-----|------|---------| | Chat | 💬 | AI assistant, EOD workflows, quick actions | | Board | 📋 | Task board organized by category | | Threads | 🧵 | Detailed view of each task's update history | | Notifs | 🔔 | EOD summaries and generated notifications | | Settings | ⚙️ | API key, model, dark mode, data management | --- ## 4. Managing Tasks (Board Tab) ### The Task Board Tasks are organized into columns by **Category** (e.g., To Do, Decision, Done). Each column shows a task count and can be collapsed by tapping the column header. **Task cards show:** - Task title - Status chip (color-coded — see Status Guide below) - Due date (turns red with ⚠️ if overdue) - Number of thread updates ### Creating a Task **Option A — Manual (Board tab):** 1. Tap **+ Add Task Manually** at the top of the Board 2. Fill in the task form fields 3. Tap **Add Task** **Option B — Via Chat:** 1. Go to Chat tab 2. Tap the **＋ Add Task** pill or type `Add task: [describe the task]` 3. The AI will create the task and add it to the board ### Editing a Task Tap any task card on the Board to open the edit modal. ### Task Fields Reference | Field | Description | Required? | |-------|-------------|-----------| | **Title** | What needs to be done | ✅ Yes | | **Category** | Which column it belongs to | ✅ Yes | | **Due Date** | When it's due | ✅ Yes | | **Status** | Current state of the task | ✅ Yes | | **Notes** | Additional context or background | Optional | | **Decision Question** | What decision needs to be made | Optional | | **Decision Options** | The possible choices (numbered) | Optional | | **Decision Maker** | Who needs to decide: Tarek, Symon, or None | Optional | | **Email Link** | Link to a related email thread | Optional | ### Status Guide | Status | Color | Meaning | |--------|-------|---------| | **Open** | Blue | Task is pending, not yet started | | **In Progress** | Amber | Actively being worked on | | **Decision** | Purple | Waiting on a decision before proceeding | | **Done** | Green | Completed | ### Decision Fields Use these three fields together for tasks that require a decision: - **Decision Question** — e.g., "Should we renew the lease for Unit 302?" - **Decision Options** — e.g., "1. Renew for 1 year\n2. Go month-to-month\n3. Let expire" - **Decision Maker** — Select **Tarek** or **Symon** to route it to the right person's EOD > **Tip:** You can also use **✨ AI Suggest** inside a task's Thread to generate decision options automatically — including context from all logged updates. --- ## 5. Tracking Work (Threads Tab) Every task has a **Thread** — a log of all updates, AI responses, and EOD entries for that task. ### Thread List View The list shows all tasks with a **sort bar** at the top: **Sort options** (tap to apply; tap again to flip direction ↑/↓): - **Last Updated** — tasks with the most recent update appear first *(default)* - **Due Date** — tasks sorted by deadline - **Status** — sorted by task status (open → in-progress → decision → done) Each card displays: - Task title and status chip - Due date and category - **🧵 N updates** chip on the lower left - **Gmail logo** beside the update count — tap to open the linked email thread (only shown if an email link is set) - **Time-ago label** in the lower right (e.g., "3m ago", "2h ago", "Yesterday") showing when the thread was last updated Tap any task to open its full thread. ### Thread Detail View Once inside a thread, the **task title header is pinned** at the top so it stays visible as you scroll through a long thread. The **input area is pinned at the bottom** (above the tab bar), just like the Chat tab. **Header buttons (top right):** - **🔮** — Opens a Quick Decision modal instantly from wherever you are in the thread. AI generates fresh options based on the full thread context. No scrolling needed. - **✏️** — Opens the task edit form **1. Overview Block** - An AI-generated 2–3 sentence summary of the task's current status - Tap **↻ Refresh** to regenerate the summary from recent updates **2. Decision Block** - Shows the saved decision question and options - Use **✨ AI Suggest** to generate 2–3 decision options — the AI reads the task details **plus the full thread history** and any previously saved options, so suggestions stay relevant as the task evolves - Use **➕ Add More** to append additional AI-suggested options (avoids repeating existing or saved options) - Check/uncheck options, then tap **Save Checked** to save to the task **Quick Decision Modal (🔮)** Tap **🔮** in the header at any time — even deep in a long thread — to open a floating panel that: 1. Immediately generates new decision options using the full thread history as context 2. Shows options as checkboxes (same as the inline Decision Block) 3. Supports **➕ Add More** and **Save Checked** 4. Closes automatically after saving, or tap Cancel / outside to dismiss without saving **3. Updates List** Each entry is color-coded by its source: | Color | Tag | Source | |-------|-----|--------| | 🟠 Orange left border | `EOD` | Logged during an EOD workflow | | 🟢 Green left border | `UPDATE` | Manually logged or via Update Task | | 🔵 Blue left border | `AI ASSIST` | AI Assist mode response | | 🟡 Amber left border | `AGENDA` | Logged during an Agenda workflow | The source tag and timestamp appear together at the bottom of each entry. All timestamps are displayed in **US Central time**. ### Log Mode vs Assist Mode At the bottom of a thread, you'll see two mode buttons and a small mode indicator label: - **📝 Log** — Type a plain update. It saves directly to the thread (no AI). Label shows: *📝 Log Mode* - **✨ Assist** — Chat with AI about this specific task. AI has full context of the task details and thread history. Its response is saved to the thread. Label shows: *✨ AI Assist* The label always shows which mode is active so you know what will happen when you hit send. --- ## 6. Daily Workflows (Chat Tab) The Chat tab is the main workspace. Use the **pill buttons** at the bottom for quick actions. ### Pill Buttons Quick Reference | Pill | What it does | |------|-------------| | 🏆 **Accomplishments** | Log something you completed today | | ＋ **Add Task** | Create a new task via AI chat | | ✏️ **Update Task** | Route a quick update to the right task | | 📋 **Agenda** | Generate daily follow-up questions for all open tasks | | 📊 **EOD: Tarek** | Generate end-of-day report for Tarek | | 📊 **EOD: Symon** | Generate personal end-of-day summary | | 🗑️ **Clear Chat** | Clear chat history (asks for confirmation) | --- ### Logging an Accomplishment Use this when you've completed something that wasn't a formal tracked task, or when you want it to appear in your EOD Accomplishments section. 1. Tap **🏆 Accomplishments** 2. Type your accomplishment naturally: > `Accomplishment: Submitted the lease renewal for Unit 302 — for the downtown tower managed by Tarek` 3. Tap Send 4. The AI extracts a short title and notes, creates a **Done task** with today's date 5. Confirmation: `✅ Accomplishment logged: Submitted Lease Renewal Unit 302` --- ### Adding a Task via Chat 1. Tap **＋ Add Task** 2. Describe the task: > `Add task: Follow up with insurance broker about the new policy renewal, due May 30` 3. The AI creates the task on the Board automatically --- ### Updating a Task via Chat 1. Tap **✏️ Update Task** 2. Describe what happened: > `Update task: Called the plumber — confirmed repair is scheduled for Thursday` 3. The AI identifies which task this belongs to and logs the update to its thread --- ### Running the Agenda (Daily Check-In) Use **Agenda** at the start of your day to get focused action items. 1. Tap **📋 Agenda** 2. AI generates numbered follow-up questions for each open task 3. An amber banner appears: *"📋 Agenda active — reply with numbered answers"* 4. Reply with numbered answers: > `1. Sent follow-up email 2. Waiting on approval 3. Scheduled for tomorrow` 5. Each answer is logged to its task's thread with an AGENDA tag 6. Confirmation: `✅ Logged 3 agenda updates to task threads` --- ### Running EOD: Tarek Use this at the end of the day to generate a formal report for Tarek. The report includes: - **Accomplishments** — Tasks completed today - **Task Updates** — All open tasks with your status updates - **Decisions Required** — Tasks assigned to Tarek that need his decision (with A/B/C options) **Steps:** 1. Tap **📊 EOD: Tarek** 2. AI asks numbered questions for each open task 3. A purple banner appears: *"📊 EOD: Tarek — reply with numbered answers"* 4. Reply with numbered answers: > `1. Sent the proposal 2. Still waiting on client feedback, follow up tomorrow 3. Meeting confirmed for Monday` 5. App logs each answer to the corresponding thread 6. AI generates the formatted EOD notification for Tarek (appears in **🔔 Notifs**) --- ### Running EOD: Symon (Personal) Same as above but creates a personal summary where **Decisions Required** shows tasks assigned to **Symon** only (not Tarek). Good for your own end-of-day review. --- ## 7. Reading Notifications (Notifs Tab) Notifications are generated automatically by EOD workflows. **Each notification shows:** - Timestamp (in US Central time) - Formatted EOD summary with sections: Accomplishments / Task Updates / Decisions Required - **📋 Copy Text** — copies the plain text to clipboard (for pasting into email, WhatsApp, etc.) - **Mark Read** — dims the notification once you've reviewed it ### For Tarek — How to Read Your EOD When Symon sends you an EOD, look for it in the **🔔 Notifs** tab. The format is: ``` EOD Update — [Day, Month Date] **Accomplishments** • [Completed task] — [context] **Task Updates** • **[Task name]** — [Symon's update] **Decisions Required** • **[Task name]** — [Decision question] A. [Option 1] B. [Option 2] C. [Option 3] ``` Reply to Symon with your decisions (e.g., "3A, 4B") so he can log them. --- ## 8. Settings & Data ### Settings Overview | Setting | Description | |---------|-------------| | **API Key** | Your Anthropic API key for AI features | | **AI Model** | Claude Sonnet (recommended), Opus (powerful), Haiku (fast) | | **Dark Mode** | Toggle light/dark theme | | **Usage** | Shows estimated API cost + token counts | | **Reset Counter** | Resets the token usage counter | | **Clear All Data** | ⚠️ Deletes ALL tasks, threads, notifications, and chat history | > **Warning:** "Clear All Data" is permanent and cannot be undone. Use only if you need a fresh start. ### Customizing Categories (Board Columns) 1. Go to **Board** tab 2. Tap the **⚙️** button at the top right 3. Edit category names, change colors, or add new columns 4. Tap **Save Categories** --- ## 9. Quick Reference — Keyboard / Chat Commands Type these directly in the Chat input: | Command | What happens | |---------|-------------| | `Accomplishment: [text]` | Creates a Done task with today's date | | `Add task: [description]` | AI creates a new task on the Board | | `Update task: [description]` | AI routes the update to the right task thread | | Any other message | General chat with Claude AI | **Voice Input:** Tap the 🎤 mic button to dictate. Recording stops automatically after 30 seconds of silence. Tap again to stop manually. --- ## 10. Role-Specific SOP ### Symon — Daily Routine **Morning:** 1. Open Chat tab → tap **📋 Agenda** 2. Review AI-generated action items for each open task 3. Reply with numbered answers to log morning status to threads **Throughout the day:** - Use **✏️ Update Task** to log progress as things happen - Use **🏆 Accomplishments** when you complete something - Open **Threads** tab to check/update specific task threads using Assist mode **End of Day:** 1. Chat tab → tap **📊 EOD: Tarek** 2. Reply to AI's questions with numbered status updates 3. Check **🔔 Notifs** tab — copy the EOD summary if you need to forward it to Tarek separately 4. Optionally run **📊 EOD: Symon** for your own personal record --- ### Tarek — How to Use the App 1. Open the **🔔 Notifs** tab to read EOD summaries from Symon 2. Look for **Decisions Required** section — each item has A/B/C options 3. Reply to Symon with your decisions 4. You can also open **📋 Board** to see all tasks at a glance 5. Tap any task to open its thread and read the full update history --- ### Other Team Members 1. **Viewing tasks:** Go to **📋 Board** to see all active tasks and their statuses 2. **Viewing thread history:** Go to **🧵 Threads**, tap any task to see its full update log 3. **Logging an update:** Inside a thread, use **📝 Log** mode to add a plain update 4. **Using AI assist:** Inside a thread, switch to **✨ Assist** mode and ask AI questions about the task --- ## 11. Tips & Best Practices - **Be specific with updates** — the more context you give, the better AI summaries and decision suggestions will be - **Set decision makers early** — assign Tarek or Symon to tasks that need decisions so they appear in the right EOD - **Use Accomplishments daily** — logging done work ensures it shows up in Tarek's EOD report - **Keep due dates current** — overdue tasks show a warning and affect EOD sorting - **Use voice input** for quick updates — just tap 🎤 and speak naturally - **Run Agenda every morning** — it keeps threads updated and surfaces tasks that need attention - **Re-run AI Suggest as tasks progress** — the more updates logged on a thread, the more accurate and relevant the decision options will be --- *TMG Ops Log is built for The Morshed Group. For technical support, contact Symon.*
